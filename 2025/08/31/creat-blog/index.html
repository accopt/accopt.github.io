部署到网上

完成你的博客内容和配置后，最后一步是部署到线上，这一步的目标是将你在本地用 Hexo 生成的静态网页文件，上传到服务器或托管平台上，让全世界都能通过 URL 访问你的博客。

我们将以最常用、最简单的 GitHub Pages 作为部署平台进行讲解，因为它对个人博客是完全免费的。

这一步主要分为两个部分：配置和部署。

第一部分：配置 GitHub Pages 仓库

  1. 创建 github 账号
    Github官网

  2. 创建新的代码仓库
    登录你的 GitHub 账号，点击右上角的 + 号，选择 New repository。

  • Repository name（仓库名）： 这是最关键的一步，仓库名必须是 你的GitHub用户名.github.io。例如，如果你的用户名是 acoppt，那么仓库名就必须是 acoppt.github.io。这个特殊的命名规则是 GitHub Pages 识别个人博客网站的依据。

  • Description（描述）： 可以不填，或者填写一些简单的描述，比如 My personal blog。

  • Public/Private： 必须选择 Public（公开），因为 GitHub Pages 的免费服务只支持公开仓库。

  • Add a README file： 勾选此项，方便初始化仓库。

最后，点击 Create repository。

第二部分 配置 Hexo 并一键部署

  1. 安装 Git 部署插件

  1. 配置博客的部署信息
    打开博客根目录的 _config.yml 文件，找到 deploy 字段，并修改为以下内容。

  • type: git：指定使用 Git 进行部署。

  • repo：非常重要。在这里，你需要填写你刚才创建的 GitHub 仓库的 HTTPS 地址。请务必将 你的GitHub用户名 替换成你自己的用户名。

  • branch：这是你博客内容所要推送到的分支。GitHub 新建的仓库默认分支是 main。如果你创建时选择了 master，请将这里也改为 master。
