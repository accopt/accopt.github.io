  2. try…catch vs .then().catch()

特性 try…catch .then().catch()
处理对象 同步代码（立刻执行，没有等待） 异步代码（需要等待，比如网络请求）
错误类型 同步错误（即时抛出，例如：语法错误、调用不存在的函数） 异步错误（Promise 拒绝，例如：网络中断、服务器返回错误）
执行时机 立即执行并捕获错误 将来某个时间点执行，处理异步任务的结果
工作原理 程序执行时立即监测代码块，如果出错则立即跳转到 catch 发起任务后程序继续执行，等待任务完成（成功或失败），再根据结果调用相应的方法
通俗比喻 打电话：拨出号码，如果没打通立刻就知道。 发短信：发送后可以去做别的事，等待对方回复或系统通知失败。

以上涉及到同步和异步的概念。

特性 同步（Synchronous） 异步（Asynchronous）
执行方式 串行执行，一次只做一件事。 并行执行，多个任务可以同时进行。
“等待”状态 程序会阻塞（停滞），直到任务完成。 程序不会阻塞，可以继续执行后续任务。
效率 对于耗时任务效率低，因为它会导致整个程序“卡住”。 对于耗时任务效率高，因为它能充分利用时间，不让程序闲置。
常见场景 简单的数学计算、变量赋值。 网络请求、数据库查询、文件读写。

  1. 文件(.txt)上传之后如何能够自动识别文件中的内容并同步在文本域中

原生的浏览器 API，文件阅读器 FileReader API

  • new FileReader(): 创建一个 FileReader 实例。

  • reader.onload: 监听文件读取成功事件。当文件读取完成后，会触发这个回调函数，event.target.result 包含了读取到的文件内容。

  • reader.onerror: 监听文件读取失败事件。

  • reader.readAsText(file, ‘UTF-8’): 以文本格式异步读取文件内容，并指定编码为 UTF-8。

  • filter 遍历数组中的每个元素
  • split 根据指定的分隔符，将一个字符串分割成一个数组

  1. CDN是什么呢？

  • 是什么：
    CDN（Content Delivery Network，内容分发网络）它通过在全球各地部署节点服务器，将网站的静态资源（如图片、视频、CSS、JavaScript等）存储在这些节点上。当用户访问网站时，CDN会根据用户的地理位置、网络状况等因素，自动选择离用户最近的节点服务器来提供资源，从而减少数据传输距离，降低网络延迟，提高资源加载速度，同时也能减轻源服务器的负载，提升网站的稳定性和访问体验。
    通俗的讲 ，CDN就像一个“网络快递中转站”,在世界各地放一堆“缓存服务器”，提前把常用的内容存到离你最近的那个,这样你访问的时候，不用跑去远方，而是直接从附近的服务器拿，速度就快了。

  • 原理：

    • 用户请求网页 → 先被智能调度系统分配 → 找离用户最近的 CDN 节点。

    • 如果这个节点有缓存好的内容 → 直接返回给用户。

    • 如果没有缓存 → 它会去源站拿一份，再保存下来给后续用户用。

  • 有什么作用呢？

    • 加速访问：网页、视频加载更快，减少卡顿。

    • 减轻源站压力：大部分请求在 CDN 节点解决，源站不用承受所有流量。

    • 防攻击：CDN 可以在前面挡一部分恶意流量，比如 DDoS 攻击。

    • 高可用性：源站挂了，CDN 节点上还有缓存，用户还能访问一段时间。
